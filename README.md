## Сервис отправки уведомлений пользователям

Внешняя система создает уведомление для отправки пользователям посредством вызова http метода POST /v1/notification.
Данный сервис сохраняет нотификацию в БД, через определенный промежуток времени ищет в БД неотправленные нотификации
и отправляет их различным способом. Планируемое кол-во экземпляров приложения в продакшене - 3 

## Что нужно сделать

1. В сущность нотификации нужно добавить поле - статус отправки, может принимать значения NEW, SENT
2. При создании нотификации нужно добавить обязательное поле - способ отправки, может принимать значения RABBITMQ, KAFKA, REST
3. При создании нотификации нужно передавать и сохранять список email адресов пользователей, кому отправлять уведомление
4. Добавить валидацию входящих данных при создании нотификации
5. Добавить сервис, который по расписанию - 1 раз в 10 мин запрашивает из БД все неотправленные нотификации и выполняет их отправку.
6. Отправка должна осуществляться способом, который указан у нотификации. Сервисы отправки реализованы имплементацией интерфейса NotificationSender, фактическую отправку делать не нужно - достаточно сделать так, что бы нотификация считалась отправленной.
7. Сделать так, что бы тест NotificationControllerTest проходил успешно
8. В текущем коде есть примеры плохого кода, желательно их исправить
9. Будет плюсом если на добавленную функциональность будут тесты
10. Доработки должны быть выполнены в виде пулл реквеста к этому репозиторию
